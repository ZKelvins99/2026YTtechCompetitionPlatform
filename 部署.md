## 二、本地打包

### 2.1 后端（Maven）

在 **PowerShell** 中执行（注意 `JAVA_HOME` 与参数引号）：

```powershell
$env:JAVA_HOME = "D:\JDK21"
cd D:\code_kj\2026YTtechCompetitionPlatform\springBoot

& "D:\apache-maven-3.8.5\bin\mvn.cmd" clean package `
  "-Dmaven.test.skip=true" `
  -s "D:\apache-maven-3.8.5\conf\settings.xml" `
  "-Dmaven.repo.local=D:\apache-maven-3.8.5\maven_lib"
```

**产物路径：**

```text
springBoot\ruoyi-admin\target\ruoyi-admin.jar
```

也可直接运行：`springBoot\bin\package.bat`（需先设置 `JAVA_HOME=D:\JDK21`）。

### 2.2 前端（Vite）

```powershell
cd D:\code_kj\2026YTtechCompetitionPlatform\web
npm install          # 首次或依赖变更时
npm run build:prod
```

**产物路径：** `web\dist\`

生产环境 API 前缀为 `/prod-api`（见 `web\.env.production`）。

### 2.3 组装发布包（建议）

```powershell
$root = "D:\code_kj\2026YTtechCompetitionPlatform"
$pkg  = "$root\deploy\packages"
New-Item -ItemType Directory -Force -Path $pkg | Out-Null

Copy-Item "$root\springBoot\ruoyi-admin\target\ruoyi-admin.jar" $pkg
Copy-Item "$root\deploy\packages\ry.sh" $pkg -Force
Copy-Item "$root\deploy\config\application-prod.yml" $pkg
Compress-Archive -Path "$root\web\dist\*" -DestinationPath "$pkg\frontend-dist.zip" -Force
```

仓库内已保留：

- `deploy\config\application-prod.yml` — 生产配置模板  
- `deploy\config\nginx-ruoyi.conf` — Nginx 站点配置模板  
- `deploy\packages\ry.sh` — 服务器启停脚本（LF 换行）

## 四、服务器部署步骤

### 4.1 首次环境准备（已完成可跳过）

**Yum 阿里云镜像 + EPEL：**

```bash
curl -fsSL -o /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo
curl -fsSL -o /etc/yum.repos.d/epel.repo http://mirrors.aliyun.com/repo/epel-7.repo
yum clean all && yum makecache
```

**OpenJDK 17（华为云镜像）：**

```bash
mkdir -p /opt/ruoyi/{backend,frontend,uploadPath,logs,config}
cd /usr/local/src
curl -fsSL -o openjdk-17.tar.gz https://mirrors.huaweicloud.com/openjdk/17.0.2/openjdk-17.0.2_linux-x64_bin.tar.gz
tar xzf openjdk-17.tar.gz -C /usr/local/
ln -sf /usr/local/jdk-17.0.2 /usr/local/jdk17
echo 'export JAVA_HOME=/usr/local/jdk17' > /etc/profile.d/jdk17.sh
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> /etc/profile.d/jdk17.sh
```

**Redis 7.2.7 + Nginx：** 见此前运维记录；Redis 仅监听 `127.0.0.1`。

**SELinux（Nginx 反向代理必做）：**

```bash
setsebool -P httpd_can_network_connect 1
```

**系统字体（验证码 `/captchaImage` 必装）：**

若依登录页验证码使用 Kaptcha + Java AWT 渲染文字，CentOS 最小化安装通常无字体，会导致 `FontConfiguration.head is null` 及 `Handler dispatch failed: java.lang.InternalError`。

```bash
yum install -y fontconfig dejavu-sans-fonts liberation-fonts
fc-cache -fv

# Kaptcha 默认字体为 Arial、Courier，映射到已安装字体（可选但建议）
cat > /etc/fonts/local.conf << 'EOF'
<?xml version="1.0"?>
<!DOCTYPE fontconfig SYSTEM "fonts.dtd">
<fontconfig>
  <alias>
    <family>Arial</family>
    <prefer><family>Liberation Sans</family></prefer>
    <prefer><family>DejaVu Sans</family></prefer>
  </alias>
  <alias>
    <family>Courier</family>
    <prefer><family>Liberation Mono</family></prefer>
  </alias>
</fontconfig>
EOF
fc-cache -fv
```

`ry.sh` 中 **必须** 包含 JVM 参数 `-Djava.awt.headless=true`（仓库 `deploy/packages/ry.sh` 已配置），否则 Kaptcha 在无图形界面服务器上会报 `InternalError`。

安装或修改字体配置后需 **重启后端**：`cd /opt/ruoyi/backend && ./ry.sh restart`。

### 4.2 每次发版：更新前端

```bash
cd /opt/ruoyi/frontend
rm -rf ./*
unzip -o /opt/ruoyi/frontend-dist.zip -d /opt/ruoyi/frontend
# 若 unzip 报 Windows 路径分隔符警告，可忽略，确认 index.html 存在即可
test -f /opt/ruoyi/frontend/index.html && echo OK
```

### 4.3 每次发版：更新后端

```bash
# 上传 jar 后
sed -i 's/\r$//' /opt/ruoyi/backend/ry.sh   # Windows 上传脚本时必做
chmod +x /opt/ruoyi/backend/ry.sh

cd /opt/ruoyi/backend
./ry.sh stop
# 覆盖 ruoyi-admin.jar 后再启动
./ry.sh start
tail -f logs/stdout.log
```

看到日志中出现 **「若依启动成功」** 且端口 `8080` 监听即正常。

### 4.4 Nginx

配置文件：`/etc/nginx/conf.d/ruoyi.conf`（模板见 `deploy/config/nginx-ruoyi.conf`）。

```bash
# 禁用默认站点，避免冲突
mv /etc/nginx/conf.d/default.conf /etc/nginx/conf.d/default.conf.bak 2>/dev/null
nginx -t && systemctl reload nginx
```

**验证：**

```bash
curl -I http://127.0.0.1/                          # 前端 200
curl -I http://127.0.0.1/prod-api/captchaImage     # 代理 API 200
curl -I http://127.0.0.1:8080/captchaImage         # 后端直连 200
# 验证码内容（应含 "操作成功" 与 base64 图片字段 img）
curl -s http://127.0.0.1/prod-api/captchaImage | head -c 80
```

浏览器访问：`http://128.128.100.16/`

## 五、配置说明（变更后部署必看）

### 5.1 生产覆盖配置 `application-prod.yml`

路径：`/opt/ruoyi/config/application-prod.yml`

当前主要内容：

- `ruoyi.profile` → `/opt/ruoyi/uploadPath`（上传文件目录）
- 日志级别 `info`
- 关闭 devtools 热部署
- `referer.allowed-domains` 含服务器 IP

**修改后无需重新打 jar**，重启后端即可：

```bash
cd /opt/ruoyi/backend && ./ry.sh restart
```

### 5.2 数据库 / Redis（在 jar 内，改则需重打 jar）

| 配置 | 文件 | 说明 |
|------|------|------|
| Oracle 连接 | `application-druid.yml` | 改 URL/账号后需 Maven 重新打包 |
| Redis | `application.yml` | 默认 `localhost:6379`，无密码 |
| 服务端口 | `application.yml` | 默认 `8080` |

若仅改 Oracle/Redis，编辑源码中 yml → 重新执行 **第二节 Maven 打包** → 上传 jar → `./ry.sh restart`。

### 5.3 前端 API 地址

修改 `web\.env.production` 中 `VITE_APP_BASE_API` 后，需重新 `npm run build:prod` 并上传前端 zip。

### 5.4 Nginx 反向代理规则

- 静态：`/` → `/opt/ruoyi/frontend`
- API：`/prod-api/` → `http://127.0.0.1:8080/`（去掉 `/prod-api` 前缀）
- 上传资源：`/profile/` → 后端 `/profile/`
- 上传大小：`client_max_body_size 100m`（批量导入等，未配置时 Nginx 默认 1m 会返回 **413**）

## 七、常见问题

| 现象 | 处理 |
|------|------|
| `./ry.sh: bad interpreter` | `sed -i 's/\r$//' /opt/ruoyi/backend/ry.sh` |
| `/prod-api` 返回 502 | ① SELinux：`setsebool -P httpd_can_network_connect 1`（nginx 日志 `Permission denied`）② 后端未就绪：重启后约 10s 内会 `Connection refused`，等 `./ry.sh` 输出 `Backend is ready` 或 `curl` 验证码成功后再访问 ③ 确认 `./ry.sh status` 与 `ss -tlnp \| grep 8080` |
| Maven 报 target 17 无效 | 设置 `$env:JAVA_HOME="D:\JDK21"` |
| MCP 上传报路径不允许 | 在 mcp.json 增加 `--allowed-local-paths` 并重启 MCP |
| 后端连不上 Oracle | 检查 100.16 到 128.128.9.3:1521 网络与账号 |
| 仅本机访问 Redis | 已绑定 127.0.0.1，勿改 bind 除非明确需要 |
| 登录页报 `Handler dispatch failed` / `/captchaImage` 500 | ① `ry.sh` 的 `JVM_OPTS` 含 `-Djava.awt.headless=true` 并重启；② 执行 **4.1 系统字体** 安装与 Arial 别名；日志常见 `FontConfiguration.head is null` |
| 批量导入 / 上传 **413** | `ruoyi.conf` 设置 `client_max_body_size 100m` 后 `nginx -t && systemctl reload nginx`；`application-prod.yml` 中 `multipart` 上限同步为 100MB |
